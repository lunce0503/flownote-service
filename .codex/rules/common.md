# 공통 규칙

- 하위 프로젝트 명령이 별도 디렉터리를 요구하지 않으면 저장소 루트에서 작업한다.
- 편집 전에 영향을 받는 하위 프로젝트를 식별한다.
- 변경은 작고, 국소적이며, 되돌릴 수 있게 유지한다.
- 기존 헬퍼, API, 설정 방식, 폴더 컨벤션을 우선한다.
- 명시 요청이 없으면 커밋, 푸시, 리셋, 삭제, 관련 없는 작업 덮어쓰기를 하지 않는다.
- 설정 이름을 이해하는 데 필요한 범위를 넘어서 비밀값을 읽거나 노출하지 않는다.
- 생성물, 의존성 폴더, 빌드 산출물, 캐시, 업로드 파일은 편집하지 않는다.
- 가능하면 구조화된 파서와 프레임워크 API를 사용한다.
- 구현 후 가장 좁고 의미 있는 명령으로 검증한다.
- 작업 종료 전 저장소 루트에서 `docker compose up -d --build`를 실행해 통합 Docker 빌드와 백그라운드 실행을 검증한다.
- 실행하지 못한 명령은 이유와 함께 기록한다.
