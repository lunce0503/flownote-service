---
name: deploy-server
description: 서버 배포는 개발자가 작성한 코드를 운영 환경에 배포하여 사용자에게 제공하는 과정입니다. 이 과정은 코드의 안정성, 성능, 보안 등을 고려하여 최적의 방식으로 진행되어야 합니다.
---

1. 배포 전에 코드를 빌드하고 테스트하여 안정성을 확인한다.
2. 배포 스크립트나 도구를 사용하여 자동화된 배포 프로세스를 구축한다.
3. 배포 후 모니터링을 통해 시스템의 상태를 확인하고, 문제가 발생하면 신속하게 대응한다.
4. 롤백 계획을 마련하여 배포 실패 시 빠르게 이전 버전으로 복구할 수 있도록 한다.
5. 배포 로그를 기록하여 문제 발생 시 원인을 분석할 수 있도록 한다.
