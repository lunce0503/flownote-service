---
name: feature-planning
description: 기능 작업 계획은 새로운 기능을 개발하기 전에, 해당 기능이 시스템에 미치는 영향과 필요한 변경 사항을 분석하여, 효율적이고 효과적인 개발을 위한 로드맵을 만드는 과정입니다. 이 계획은 기능 구현에 필요한 리소스, 시간, 그리고 잠재적인 리스크를 식별하여, 개발 팀이 명확한 목표와 방향성을 가지고 작업할 수 있도록 돕습니다.
---

1. 영향받는 하위 프로젝트를 식별한다.
2. 기존 패턴과 유사 구현을 검색한다.
3. 변경 가능성이 있는 파일과 계약을 나열한다.
4. 가장 작은 구현 경로를 정의한다.
5. 검증 명령을 선택한다.
6. 보안, DB, UX 리스크를 기록한다.
