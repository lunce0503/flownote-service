# 품질 게이트 체크리스트

- 영향받는 하위 프로젝트를 식별했다.
- 구현이 기존 로컬 패턴과 맞는다.
- 비밀값, 인증 정보, 덤프, 로컬 전용 값을 추가하지 않았다.
- 생성 폴더와 의존성 폴더를 건드리지 않았다.
- 관련 lint/build/test 명령을 실행했거나, 생략 사유를 명확히 남겼다.
- 저장소 루트에서 `docker compose up -d --build`를 실행했거나, 생략 사유를 명확히 남겼다.
- 파일을 수정한 작업이면 Vercel production 및 영향받는 Railway 서비스 배포를 실행했거나, 실행하지 못한 이유와 다음 배포 명령을 남겼다.
- `report/` 전용 보고서 작업이면 Docker와 클라우드 배포를 생략하고 파일 존재/내용 확인으로 검증했다.
- 호출부 전반에서 API 계약을 확인했다.
- 사용자에게 보이는 변경에서 오류, 빈 상태, 로딩 상태를 고려했다.
- DB 변경에서 마이그레이션과 데이터 호환성을 고려했다.
- 재현 가능한 버그는 `logs/bugs/`에, 리뷰/감사 산출물은 `report/`에 남겼거나 생략 사유를 정리했다.
- 최종 응답에 변경 범위, 하위 프로젝트 검증 결과, `docker compose up -d --build` 결과, 클라우드 배포 결과를 포함했다. `report/` 전용 보고서 작업이면 Docker/클라우드 배포 생략 사유를 포함했다.
