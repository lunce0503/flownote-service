# Flownote Codex 하네스

이 디렉터리는 Codex용 프로젝트 로컬 하네스 자산을 담는다. 루트 `AGENTS.md`가 항상 적용되는 기본 지침이며, `.codex/` 파일들은 재사용 규칙, 프롬프트, 체크리스트, 메모리, 워크플로우 참조를 제공한다.

## 구성

- `config.example.toml`: 이 프로젝트에서 참고할 Codex 프로필 설정 예시.
- `rules/`: 공통 및 스택별 엔지니어링 규칙.
- `skills/`: 안정화 후 사용자 레벨 Codex 스킬로 승격할 수 있는 프로젝트 로컬 스킬 초안.
- `prompts/`: 계획, 리뷰, 디버깅, 품질 게이트용 재사용 프롬프트.
- `checklists/`: 관심사별 검증 체크리스트.
- `memories/`: 장기 프로젝트 컨텍스트 후보.
- `mcp/`: MCP 설정 메모와 예시.
- `harness/`: 워크플로우, 라우팅, 검증 매트릭스.
- `../logs/`: 버그 재현, 운영 증상, 실행 로그 요약.
- `../report/`: 리뷰, 감사, 배포 결과, 큰 작업 요약.

## 운영 모델

이 하네스는 가벼운 시스템으로 사용한다.

1. `CLAUDE.md`는 필수 행동 규칙을 정의한다.
2. `rules/`는 스택별 선호와 규칙을 담는다.
3. `prompts/`는 반복 워크플로우를 표준화한다.
4. `checklists/`는 검증 누락을 줄인다.
5. `memories/`는 안정적인 프로젝트 지식을 보존한다.
6. `logs/`와 `report/`는 대화 밖에서도 추적해야 하는 증상과 결과를 보존한다.

이 디렉터리에 비밀값, 토큰, 로컬 인증 정보, 운영 데이터, 비공개 환경 변수 값을 저장하지 않는다.
